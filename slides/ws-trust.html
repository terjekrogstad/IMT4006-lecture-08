WS-Trust

  • Participants needs to determine whether or not they can trust security credentials exchanged
  • Defines additional primitives and extensions for issuing, exchanging and validating security tokens
  • Typical process
    • Service specifies required claims using WS-policy
    • A SOAP-request arrives at a service, WS-Trust requries that the message prove a set of claims, e.g. name, key and permission
